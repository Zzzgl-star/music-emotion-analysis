# 多模态音乐情感分析与推荐系统

基于音频特征（Spotify）和中文歌词文本的多模态音乐情感分析系统。融合传统机器学习与深度学习，实现音乐情感四分类（happy/sad/calm/energetic）及个性化音乐推荐。


## 项目简介

项目是数据挖掘课程的一个课程项目，从头到尾自己独立完成。核心思路是把音乐的音频特征和歌词文本结合起来做情感分析，然后基于分析结果做一个音乐推荐系统。

整个项目分几个部分：数据采集（Spotify音频特征 + 中文歌词）、特征提取、模型训练（对比了传统机器学习和深度学习）、多模态融合，最后用Flask搭了一个简单的Web推荐界面。

** 三个关键点：**
- 多模态融合后准确率到99.7%
- 做了四种推荐策略（情感匹配、渐进式调节、多样性增强、情境感知）
- 对比下来发现小数据集上传统集成模型比深度学习效果好，训练时间能缩短70%


## 系统架构

```
数据层 -> Spotify音频特征(32000首) + 中文歌词(3643首)
   |
特征层 -> 9维音频特征 + 10维文本特征 + HSK复杂度分析
   |
模型层 -> 集成学习(0.997) vs BERT+CNN-LSTM(~0.78)
   |
融合层 -> 动态权重多模态融合(音频54.1% + 歌词45.9%)
   |
应用层 -> Flask Web推荐系统
```


## 数据集

| 数据集 | 来源 | 规模 |
|--------|------|------|
| 音频特征 | Spotify Web API (GitHub开源数据) | 32000首，9维特征 |
| 中文歌词 | 和鲸社区 | 3643首，23个字段 |
| 平衡后数据集 | 智能配对（SMOTE+欠采样） | 1600首，每类400首 |

## 实验结论

1. 多模态融合比单模态提升5-10%
2. 小数据集（<5000样本）上传统集成方法优于深度学习
3. 音频特征在音乐情感识别中起主导作用
4. 动态权重融合策略有效（音频54.1% + 歌词45.9%）
